# [10장] 게임 수학과 벡터 심화 예상 문제

## 문제 1. 내적(Dot Product)을 이용한 위치 판별
내 캐릭터가 발사한 화살의 진행 방향 벡터(`HitDirection`)와 맞은 적의 정면 벡터(`EnemyForward`)를 **내적(Dot Product)** 했을 때 그 결과(`DotResult`)가 **양수(>0)** 라면, 적은 어느 부위를 맞은 것인가?

1) 앞통수 (정면)
2) 등 뒤 (후방)
3) 정확한 측면 (90도)
4) 알 수 없다

## 문제 2. 외적(Cross Product)의 쓰임새
두 정규화 벡터 A와 B를 **외적(Cross Product)** 했을 때 나오는 결과는 무엇인가?

1) 두 벡터 사이의 각도(Cos 스칼라 값)
2) 두 벡터가 이루는 평면의 넓이
3) 두 벡터가 만드는 평면을 뚫고 수직(직교)으로 뻗어 나가는 제3의 3D 방향 벡터(Z축 등)
4) 두 벡터의 길이의 합

## 문제 3. 벡터의 정규화 (Normalize)
`GetActorForwardVector()` 함수가 반환하는 결과값 벡터의 가장 중요한 특징은 무엇인가?

1) 현재 캐릭터의 이동 속도(Speed) 값이 이미 곱해져 있다.
2) 벡터의 길이(Size)가 무조건 1.0으로 고정된 순수한 방향(Direction) 벡터이다.
3) 무조건 월드의 정중앙 좌표(0,0,0)를 가리키는 벡터이다.
4) 포인터 메모리로 반환된다.

## 문제 4. 외적 교환 법칙의 함정 (코드 예측)
A벡터(X축)와 B벡터(Y축)가 있을 때, 다음 두 외적 연산의 결과에 대한 설명으로 올바른 것은?

```cpp
FVector Result1 = FVector::CrossProduct(A, B);
FVector Result2 = FVector::CrossProduct(B, A);
```

1) `Result1`과 `Result2`는 완전히 동일한 Z축 벡터이다.
2) `Result1`은 위로 솟구치는 Z축 벡터라면, 순서를 뒤집은 `Result2`는 아래로 처박히는 반대 방향(-Z축) 벡터가 나온다.
3) 컴파일 에러가 발생한다.
4) 둘 다 크기가 0인 ZeroVector가 나온다.

## 문제 5. 벡터의 안전한 정규화 (빈칸 채우기)
벡터의 길이가 0일 때 강제로 정규화하면 분모가 0이 되어 크래시가 납니다. 이를 방어하고 안전하게 크기를 1로 깎아주는 내장 함수는 무엇인가?

```cpp

// 크기가 0에 가깝지 않다면 안전하게 1.0짜리 방향 벡터로 만든다.
FVector MoveDir = Velocity.[      빈 칸      ]();
```

2) `GetSafeNormal()`
3) `UnitVector()`

# 정답 및 해설

### 문제 1 정답: 2번 (등 뒤)
**해설:** 화살과 적이 같은 방향을 바라보면 두 벡터 사이의 각도가 예각(90도 미만)이 되어 내적(Cos) 값이 양수가 나옵니다. 즉 화살이 뒤통수에서 꽂혔다는 뜻입니다. (서로 마주보는 정면의 경우 둔각이 되어 음수가 나옵니다. 이를 이용해 백어택 추가 데미지를 구현합니다.)

### 문제 2 정답: 3번
**해설:** 두 3D 벡터를 외적하면 두 벡터가 구성하는 평면에 직교하는 새로운 3D 법선 벡터(Normal)를 뱉어냅니다. 이를 통해 지형의 경사로나 바닥 기울기, 왼손/오른손 좌표계의 회전 축을 구할 때 절대적으로 쓰이는 필수 연산입니다.

### 문제 3 정답: 2번
**해설:** 길이(크기)가 1.0으로 통일된 정규화(Normalized) 단위 벡터는 오염되지 않은 순수한 '방향'만을 나타냅니다. 여기에 기획자가 원하는 스피드 값을 단순 곱셈 연산(스케일링)하여 최종 이동량을 결정하게 됩니다.

### 문제 4 정답: 2번
**해설:** 외적(Cross Product)은 스칼라 곱셈과 달리 교환 법칙이 성립하지 않습니다! A x B 와 B x A 는 크기는 같지만 방향이 정확히 정반대(180도)로 뒤집힌 법선 벡터가 생성됩니다. (왼손/오른손 3D 공간 법칙)

### 문제 5 정답: 2번 (`GetSafeNormal()`)
**해설:** 일반 `Normalize()` 함수를 벡터 길이가 0(안 움직임)인 녀석에게 썼다간 0 나누기 오류로 게임이 뻗습니다. 언리얼은 이를 방어하기 위해 길이가 너무 작으면 그냥 `ZeroVector(0,0,0)`를 반환하고, 아니면 1로 만들어 반환하는 매우 튼튼한 `GetSafeNormal()` 함수를 지원합니다. 실무에서 아주 흔히 쓰입니다.
