# [10장] 게임 수학과 벡터 심화 예상 문제

## 문제 1. 내적(Dot Product)을 이용한 위치 판별
플레이어(`PlayerPawn`)가 쏜 화살이 몬스터(`Enemy`)의 등 뒤(후방)에 꽂혔는지, 아니면 앞통수(전방)에 꽂혔는지 판별하려고 한다. 
플레이어가 쏜 방향 벡터(`HitDirection`)와 몬스터가 바라보는 방향 벡터(`EnemyForward`)를 가져와 **내적(Dot Product)**을 수행했다. 
만약 **등 뒤(후방)에서 화살이 꽂혀서 두 벡터의 방향이 거의 같다면**, 내적 결과값(`DotResult`)의 범위는 어떠해야 하는가?

1) `DotResult > 0`
2) `DotResult == 0`
3) `DotResult < 0`
4) 범위로는 판별할 수 없다.

## 문제 2. 외적(Cross Product)의 쓰임새
두 개의 정규화된 방향 벡터 A와 B가 있다. 이 두 벡터를 **외적(Cross Product)** 했을 때 나오는 결과값의 수학적 특징은 무엇인가?

1) 두 벡터 사이의 각도(Cos)를 스칼라(실수) 값으로 반환한다.
2) 두 벡터를 더한 중간 지점의 좌표를 반환한다.
3) 두 벡터가 이루는 평면에 수직(직교)으로 뻗어 나가는 완전히 새로운 제3의 방향 벡터(Z축 등)를 만들어 반환한다.
4) 두 벡터의 길이를 곱한 순수 면적 값을 반환한다.

## 문제 3. 벡터의 정규화 (Normalize)
`GetActorForwardVector()`나 `GetActorRightVector()`를 호출하면 반환되는 `FVector` 객체의 **가장 중요한 특징**은 무엇인가?

1) 이동 속도(Speed) 값이 미리 곱해져 있다.
2) 벡터의 길이(Size)가 항상 **1.0**으로 셋팅된 순수한 방향(Direction)만을 나타낸다.
3) 언리얼의 절대 좌표계 영점(0, 0, 0)을 기준으로 한 월드 좌표이다.
4) 가비지 컬렉터에 의해 보호받는 포인터 타입이다.

# 정답 및 해설

### 문제 1 정답: 1번 (`DotResult > 0`)
**해설:** 화살이 날아가는 방향(HitDirection)과 적이 바라보는 방향(EnemyForward)이 **같은 방향**을 향하고 있다면, 사잇각이 예각(90도 미만)이 되므로 코사인(Cos) 연산인 내적의 결과는 무조건 **양수(>0)**가 나옵니다. 반면 앞에서 얼굴을 맞췄다면 두 벡터가 서로 마주보는 반대(둔각)를 향하므로 음수(<0)가 나옵니다.

### 문제 2 정답: 3번
**해설:** 스칼라(float) 값을 반환하는 내적과 달리, 외적은 두 벡터(예: X축과 Y축)가 만드는 평면을 뚫고 수직으로 올라오는(또는 내려가는) 새로운 **3D 방향 벡터(Z축)**를 만들어내는 공간 연산입니다. 면의 수직 방향(Normal)을 구할 때 매우 자주 쓰입니다.

### 문제 3 정답: 2번
**해설:** 언리얼에서 방향을 나타내는 대부분의 내장 Forward/Right/Up 벡터들은, 연산의 오류를 막기 위해 크기(길이)가 무조건 '1'로 고정된 **정규화(Normalized) 벡터** (또는 단위 방향 벡터)입니다. 여기에 사용자가 원하는 이동 속도나 힘을 곱해서(스케일링) 최종 이동 벡터를 만듭니다.
