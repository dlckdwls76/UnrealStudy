# 언리얼 엔진 서브시스템(Subsystem) 계층 구조

언리얼 엔진 4.22 버전부터 도입되어, 기존의 싱글톤 패턴과 매니저(Manager) 클래스들을 완전히 대체하고 있는 초강력 생명주기(Lifecycle) 자동화 시스템입니다.

## 1. 서브시스템의 상속 계층
서브시스템은 "어떤 단위와 함께 태어나고 죽을 것인가?(생명주기)"에 따라 5가지로 나뉘어 상속됩니다.

```text
UObject
 └── USubsystem (모든 서브시스템의 최상위 부모, 자동 메모리 로드/해제 관리 기능 내장)
      │
      ├── UEngineSubsystem (가장 거대한 단위)
      │    └─ 언리얼 엔진 프로세스가 켜질 때 생성되고, 완전히 종료될 때 파괴됨. 에디터 환경에서도 돌아감.
      │
      ├── UEditorSubsystem (에디터 전용)
      │    └─ 언리얼 에디터 환경에서만 동작하는 매니저. 게임 플레이 중엔 동작하지 않음.
      │
      ├── UGameInstanceSubsystem (게임 플레이 단위)
      │    └─ UGameInstance와 생명주기를 같이 함. 게임이 시작되면 생성되고 끝날 때 파괴됨. 
      │       레벨을 이동해도 데이터가 날아가지 않아 로컬 전역 데이터(스탯, 업적) 유지에 최적.
      │
      ├── UWorldSubsystem (레벨 단위)
      │    └─ 특정 레벨(World)이 로드될 때 생성되고, 다른 레벨로 넘어가면 자동 파괴됨. 
      │       해당 레벨 안에서만 유효한 NPC 스폰 매니저나 날씨 매니저 등에 적합.
      │
      └── ULocalPlayerSubsystem (플레이어 단위)
           └─ 한 컴퓨터 안의 로컬 플레이어별로 생성. 화면 분할(Split Screen) 멀티플레이 게임을 할 때, 
              1P용 UI 매니저와 2P용 UI 매니저를 각각 격리해서 관리할 때 사용.
```
