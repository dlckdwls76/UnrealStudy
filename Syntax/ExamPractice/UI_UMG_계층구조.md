# UMG (언리얼 모션 그래픽) 위젯 계층 구조

이 문서는 플레이어의 화면에 나타나는 체력바, 인벤토리, 미니맵 등 UI(User Interface)를 구성하는 UMG 클래스들의 상속 계층도입니다.

## 1. UI 최상위 기반 상속 구조
모든 UI 요소들은 결국 하나의 거대한 위젯 계층에서 파생됩니다.

```text
UObject
 └── UVisual (시각적으로 화면에 그려질 수 있는 최상위 인터페이스)
      └── UWidget (UMG 디자이너 패널에서 끌어다 쓸 수 있는 모든 UI 부품의 부모)
           ├── UUserWidget (우리가 '위젯 블루프린트'를 만들 때 부모가 되는 핵심 클래스)
           └── UPanelWidget (자신 내부에 다른 위젯들을 담을 수 있는 '그릇/패널' 위젯류)
```

## 2. 기본 컨트롤 위젯 (단일 부속품)
단독으로 기능을 수행하며, 보통 다른 위젯을 자식으로 품을 수 없는(단말) UI 컴포넌트들입니다.

```text
UWidget
 ├── UImage (이미지, 텍스처, 혹은 머티리얼을 화면에 출력)
 ├── UTextBlock (텍스트, 폰트 출력)
 ├── UProgressBar (체력바, 로딩바, 마나 등 퍼센티지 게이지 출력)
 ├── UButton (클릭, 호버 등의 이벤트를 감지하는 클릭 가능 영역)
 ├── USlider (마우스로 잡고 끌 수 있는 볼륨 조절/수치 조절 슬라이더)
 ├── UEditableText (유저가 직접 키보드로 타이핑해서 입력할 수 있는 텍스트 상자)
 └── UCheckBox (On/Off 상태를 토글할 수 있는 체크박스)
```

## 3. 패널 위젯 (UPanelWidget)
다른 UI 부품들을 자식으로 담아 차곡차곡 정렬하고 배치하는 '그릇(Container)' 역할을 합니다.

```text
UPanelWidget (내부에 1개 이상의 자식을 가질 수 있는 위젯)
 ├── UCanvasPanel (자식들을 화면 내에 자유로운 X, Y 절대 좌표로 배치)
 ├── UHorizontalBox (자식들을 가로(수평)로 나란히 밀어내며 배치)
 ├── UVerticalBox (자식들을 세로(수직)로 나란히 밀어내며 배치)
 ├── UGridPanel (표/격자 형태로 행렬을 지정하여 배치)
 ├── UScrollBox (내부에 담긴 내용이 박스 크기를 넘치면 스크롤 바를 생성)
 ├── UOverlay (자식들을 포토샵 레이어처럼 층층이 겹쳐서 배치할 때 사용)
 └── USizeBox (내부 자식 위젯이 마음대로 커지거나 작아지지 않게 크기를 강제로 고정)
```

### 💡 강사님께 어필할 핵심 포인트
* **UUserWidget의 중요성:** "엔진에서 제공하는 기본 버튼이나 텍스트를 조합해 나만의 커스텀 체력바 UI를 만들었다면, 그 체력바 위젯 블루프린트의 부모 클래스는 무조건 `UUserWidget`이 됩니다. 이를 통해 내가 만든 UI 자체가 하나의 새로운 부속품이 되어 다른 UI에 조립될 수 있는 모듈화가 가능해집니다."
