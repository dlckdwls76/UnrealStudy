# [11장] 컨테이너 자료구조 (TArray, TSet, TMap) 심화 예상 문제

## 문제 1. TMap의 Find 함수
`TMap<int32, FString> Inventory;` 선언 후, 1번 인덱스를 조회하여 아이템을 찾는 가장 정석적인 문법은?

1) `FString Item = Inventory.Find(1);`
2) `if (FString* FoundItem = Inventory.Find(1)) { ... **FoundItem ... }`
3) `FString* FoundItem = &Inventory[1];`
4) `FString FoundItem = Inventory.Get(1);`

## 문제 2. Emplace와 Add의 성능 차이
구조체나 객체를 TArray에 추가할 때, `Add` 함수 대신 **`Emplace`** 함수의 사용을 적극 권장하는 이유는 무엇인가?

1) 메모리의 가비지 컬렉션을 무시하고 강제 할당해주기 때문이다.
2) 배열의 맨 뒤가 아니라 무조건 맨 앞에 원소를 끼워 넣기 때문이다.
3) 임시 객체를 생성하고 복사(Copy)하는 과정을 생략하고, 전달받은 파라미터만으로 빈 배열 메모리 안에 객체를 직접 생성(Construct)하여 성능 낭비(오버헤드)를 줄이기 때문이다.
4) 중복된 값이 들어오는 것을 해시를 통해 원천 차단하기 때문이다.

## 문제 3. TSet의 검색 알고리즘 속도
수만 개의 데이터를 가진 `TSet`에서 `Contains` 함수를 사용해 특정 데이터가 있는지 검색할 때 걸리는 알고리즘 속도(시간 복잡도)는 어떻게 되는가?

1) O(N) (전체를 하나씩 순회)
2) O(N^2)
3) O(log N)
4) O(1) (해시를 통한 즉시 접근)

## 문제 4. TMap의 이중 검색 비효율 (코드 예측)
이 코드가 작동은 하지만 **비효율적인 이유**로 가장 알맞은 것은?

```cpp
if (MyMap.Contains(100))
{
    FString Item = MyMap[100];
    UE_LOG(LogTemp, Warning, TEXT("찾음: %s"), *Item);
}
```

1) 컴파일 오류가 발생한다.
2) `Contains`에서 내부적으로 해시 트리를 타고 검색을 완료했는데, `MyMap[100]`에서 값을 꺼내기 위해 또다시 동일한 검색을 반복(이중 검색)하여 CPU 연산을 낭비하기 때문이다.
3) `MyMap[100]`은 값을 맵에서 삭제하는 연산자이다.
4) 런타임 크래시가 발생한다.

## 문제 5. Emplace 파라미터 전달 (빈칸 채우기)
구조체 배열에 원소를 넣을 때, 복사 없이 `Emplace`에 파라미터만 툭 던지는 깔끔한 (A)의 코드는 무엇인가?

```cpp
TArray<FMyStat> Stats;

// Add는 임시 객체를 만들어 통째로 복사해서 넣는다.
Stats.Add(FMyStat(100, 200));

// 아래 함수는 구조체 매개변수(100, 200)만 쏙 던져줘서 배열 안에서 생성자를 직통으로 호출한다.
Stats.[    (A)    ](100, 200);
```

1) `Insert`
2) `AddUnique`
3) `Emplace`
4) `Push`

# 정답 및 해설

### 문제 1 정답: 2번
**해설:** `Find()` 함수는 배열처럼 값을 통째로 빼주는 것이 아니라 원본 값의 위치를 가리키는 포인터(`*`)를 줍니다. 맵 안에 그 키가 없으면 `nullptr`를 주므로, `if`문 안에서 받아 곧바로 널 체크를 거치는 것이 안전하고 가장 완벽한 C++ 패턴입니다.

### 문제 2 정답: 3번
**해설:** `Emplace`는 메모리를 추가로 낭비하며 임시 구조체 복사본을 찍어내지 않고, 매개변수만 전달받아 컨테이너 메모리 안에 알맹이를 직접 직조(Construct)해 버립니다. 최적화의 첫 단추입니다.

### 문제 3 정답: 4번 (O(1))
**해설:** 집합(`TSet`)과 딕셔너리(`TMap`)는 배열(`TArray`)과 달리 순서대로 저장되지 않습니다. 대신 해시(Hash) 알고리즘을 타기 때문에 데이터가 수십만 개여도 인덱스를 한 번에 연산해 내어 번개처럼 O(1) 속도로 찾아냅니다. 

### 문제 4 정답: 2번 (이중 검색 오버헤드)
**해설:** `Contains`도 맵 트리를 뒤지고 괄호 연산자 `[ ]`도 맵 트리를 또 뒤집니다. 실무에선 `if (FString* Item = MyMap.Find(100))` 한 줄로 쓰면 단 한 번의 검색으로 널 체크 유무와 값 빼오기를 동시에 완료할 수 있어 훨씬 빠르고 우아합니다.

### 문제 5 정답: 3번 (`Emplace`)
**해설:** `Emplace`의 진정한 위력은 구조체를 따로 만들어서 매개변수로 던져줄 필요조차 없이, 구조체 내부 생성자에서 요구하는 매개변수 값(100, 200 등)만 쉼표로 나열해 던져주면 알아서 배열 메모리 안에 쏙 만들어낸다는 점에 있습니다. C++11의 완벽한 기능입니다.
